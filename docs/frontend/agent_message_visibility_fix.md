## 背景
Agent 页面出现“后端响应正常，但前端看不到用户消息与回复”的问题。

## 排查结论
问题不是遮挡层，而是会话切换的消息加载副作用覆盖了本地消息：
- 新建会话后会触发 `currentConversationId` 变化
- 自动拉取历史消息时会用空列表覆盖本地刚发送的消息
- 流式更新依赖最后一条消息，导致内容无法显示

## 修复方案
在新建会话并立即发送消息时，跳过一次自动加载：
- 新增 `skipNextLoadRef` 标记
- 新建会话时置为 `true`
- `useEffect` 中检测并跳过一次加载

## 影响范围
仅影响 `Agent` 页面首次发消息的加载行为，历史会话加载逻辑保持不变。
