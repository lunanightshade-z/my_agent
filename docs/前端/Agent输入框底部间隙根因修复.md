## 问题现象
- 访客模式进入 Agent 页，输入框离底部仍有距离
- 首屏能看到输入框“从下往上移动”的动画

## 根因分析
- 输入框容器使用了 `framer-motion` 的 `layout`（自动布局动画）。
- 首屏加载时（字体/内容尺寸/滚动条等）触发布局重算，`layout` 会对位置做补间动画，导致“从下往上移动”的可见效果。
- 同时 `mt-auto` + 额外 `pb-*` 会让“贴底”在不同渲染阶段看起来不稳定。

## 修复方案
- 去掉输入框外层的 `motion.div` 与 `layout`，避免首屏 FLIP 布局动画
- 主面板使用 `flex-col justify-between`，让底部输入区天然占据底部，不依赖 `mt-auto`
- 取消输入区额外的底部 padding（`pb-0`），减少视觉空隙

## 影响范围
仅影响 `frontend/src/pages/Agent.jsx` 的布局与进入动画，不影响聊天逻辑与流式响应。
