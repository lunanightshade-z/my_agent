## 问题描述
- Agent 页面在流式输出过程中，内容短暂显示后消失，直到回答结束才完整显示。

## 原因分析
- Agent 页面加载消息的副作用依赖 `conversations`，在流式阶段刷新会话列表时会触发重新拉取消息。
- 重新拉取的消息会覆盖正在流式追加的本地消息，导致最后一条流式消息被清空或替换。

## 解决方案
- 在流式阶段跳过消息拉取，避免覆盖正在进行的流式渲染。
- 流式结束后再允许消息加载与刷新。

## 变更点
- `frontend/src/pages/Agent.jsx`：在加载消息的 `useEffect` 中加入 `isStreaming` 保护并纳入依赖。

- `docker compose up -d --build` 通过。
